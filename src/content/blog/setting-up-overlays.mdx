# Setting Up the New In-Game HUD / Overlay for *Pits n' Giggles*

The latest update to **Pits n' Giggles** introduces a full in-game **HUD / overlay system**, built for F1 25 and the wider sim racing community.
These overlays provide clearer visibility, more accurate telemetry, and race-engineer-grade insights—all directly on top of your game.

## Initial Setup

Before using the HUD, make sure both **F1 25** and **Pits n' Giggles** are configured correctly.

---

## F1 25 / Game Telemetry Setup

1. Launch **F1 25**.
2. Navigate to **Settings → Telemetry Settings**.
3. Enable:
   - UDP Telemetry
   - UDP Broadcast Mode
   - Send Rate: **60 Hz**
   - **IP Address** = machine running Pits n' Giggles
4. Save settings.

Correct telemetry setup ensures the HUD receives accurate data for lap times, tyre wear, ERS, DRS, and fuel calculations.

---

## Pits n' Giggles App Setup

If you're installing the app for the first time:

> **Backlink:** [How to Install & Configure Pits n' Giggles](../../docs/setup.md)

Then:

1. Open **Settings → HUD**.
2. Enable **In-Game Overlays**.
3. Adjust **global opacity** if desired.
4. Restart the app when prompted.

The HUD currently supports **Windows** only.

---

## Configuring Overlay Position & Size

Every overlay appears as a floating, always-on-top window.

### Unlock Mode
1. Click **Unlock** in the app.
2. Drag each overlay to position it.
3. Resize them to your preference.

### Lock Mode
Click **Lock** when done.
This activates click-through mode so the HUD never interferes with gameplay.

<Image
  src={lapTimerImg}

  widths={[300, 600, 1200]}

/>

Displays:

- Current lap
- Last lap
- Best lap
- Sector times + colours

### Dynamic Sector Colour System
Sectors can change from purple → green or green → yellow based on improvements—mirroring real-world timing behavior.

**Interpretation:**
- Green = personal best
- Purple = session best
- Mixed colours = inconsistency, traffic, or tyre degradation
